# Domain Docs

This repo uses a **single-context** layout.

- **CONTEXT.md** lives at the repo root
- **ADRs** live in `docs/adr/`

## Consumer rules

- Read `CONTEXT.md` at repo root to learn the project's domain language
- Use the glossary terms consistently in code, issues, and comments
- Read `docs/adr/` for past architectural decisions before making changes
- Only create new ADRs when the decision is hard to reverse, surprising without context, and the result of a real trade-off
